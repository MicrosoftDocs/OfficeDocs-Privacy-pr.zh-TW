# <a name="create-a-request-and-define-search-settings"></a>建立要求並定義搜尋設定

**在本文** 中：瞭解如何在Priva中建立要求，以開始履行主體許可權要求。 瞭解資料擷取的搜尋設定，以及如何精簡搜尋。

在僅限檢視 (以外的主體許可權要求系統管理員角色群組中保有角色的使用者，會[看到Priva](priva-permissions.md)許可權) 可以在主體許可權要求中建立要求。 引導式程式會逐步引導您完成有關尋找資料主體個人資料的設定，並開始完成其要求的程式。

## <a name="request-types"></a>要求類型

Priva 主體權利要求支援三種不同類型的要求：

1. **存取**：提供貴組織在Microsoft 365中保存的資料主體個人資訊摘要。

2. **導** 出：提供包含資料主體個人資訊的內容專案摘要和匯出的檔案。 這些專案會在您檢閱搜尋設定所收集的資料期間檢閱並標示為 [ **已** 包含]。

3. **已標記的待處理清單**：產生在資料檢閱期間標記的任何檔案摘要，這些檔案可能需要在Priva之外採取其他動作。 例如，您可能需要根據其要求來協助刪除資料主體的個人資訊。 您可以在Priva設定中檢視包含的標籤，並為您的組織設定自訂[標籤](priva-settings.md)。

## <a name="getting-started-with-your-first-request"></a>開始使用您的第一個要求

當您開始試用或訂閱主體許可權要求時，我們會為使用預設設定的第一個要求提供簡單、現成的設定。 此設定可協助您探索主體許可權要求工作流程，並熟悉其功能。

第一次到達 [主體權利要求] 頁面時，您會在頂端看到橫幅，其中包含 **[開始**] 按鈕。 當使用者選取此按鈕時，會出現飛出視窗窗格，其中已預先填入該使用者的資訊到名稱和電子郵件欄位中，並顯示所有預設設定。

**使用您的資訊探索要求功能**：根據您自己的資訊嘗試主體權利要求，可協助您熟悉並放心地在程式的每個階段中移動。 您將瞭解預設搜尋會產生什麼結果，並可藉由調整搜尋設定來練習精簡結果。 在 [ **收集的資料]** 索引標籤上，您可以檢閱右側預覽區域中的專案，並練習修訂文字、套用標籤、輸入附注，以及標示要包含或排除最終報表的專案 (在檢 [閱主體許可權要求](subject-rights-requests-data-review.md) 的資料) 中尋找詳細資料。

- 您不需要使用您的資訊來建立第一個要求。 如果您已準備好開始要求資料主體，請將您的名稱和電子郵件地址取代為數據主體的資訊。

若要接受所有設定並建立要求，請選取 [ **建立]**。 窗格將會關閉，而您會看到新的要求列在 [ **主體權利要求** ] 頁面上。 若要在建立要求之前變更任何預設設定，請選取 **[編輯要求詳細資料**]，這會將您放入 [主體許可權要求建立精靈](#create-a-request)。

> [!NOTE]
> 您建立的任何要求都會計入您的試用版或付費訂閱配置，不論要求使用哪個資料主體的資訊。 標準 30 天資料保留期間會在要求關閉之後套用。 瞭解如何變更 [主體許可權要求的保留期限](subject-rights-requests-reports.md#retention-periods-for-reports-and-data)。

## <a name="create-a-request"></a>建立要求

請遵循下列步驟來開始履行主體權利要求：

1. 在 [Microsoft Purview 合規性入口網站](https://compliance.microsoft.com/)中，選取左側導 **覽中的 [Priva 主體權利要求**]。

2. 在畫面右上角，選取 [ **建立要求]**。

3. 在 [ **資料主體資訊** ] 頁面上，輸入資料主體的名字和姓氏和電子郵件地址。 選 **取 [新增落地** ] 以選擇資料主體的居住國家/地區。 選取可識別資料主體與貴組織 (如客戶或目前員工) 的相關選項，然後選取 [ **下一步** ] 以移至下一個步驟。

4. 在 [ **位置]** 頁面上，決定要在何處搜尋資料主體的資訊。 將每個選項旁的狀態切換開關移至 [ **開啟** ] 位置，以選擇下列其中一個或兩個位置：

   - **Exchange**：在Exchange信箱和個別或群組Teams聊天中尋找資料。 您可以選擇搜尋組織中的所有Exchange帳戶，或選取 [**選擇帳戶**] 從 **[Exchange信箱**] 飛出視窗窗格中選取個別使用者。

   - **SharePoint**：在SharePoint網站、商務用 OneDrive網站和Teams通道中尋找資料。 您可以選擇搜尋組織中的所有SharePoint網站，或選取 [**選擇網站**] 從 **[SharePoint 網站**] 飛出視窗窗格中選取個別使用者。

> [!TIP]
> 如需識別適當搜尋字詞的說明，請參閱下列主題：
> - SharePoint網站和 URL：[在SharePoint系統管理中心管理網站](/sharepoint/manage-sites-in-new-admin-center)提供如何排序和篩選網站，以及如何搜尋SharePoint網站的指引。 使用此選項來尋找要在 [**SharePoint 網站**] 飛出視窗窗格的搜尋欄位中輸入的 URL。
> - Teams聊天和頻道：[Get-Team](/powershell/module/teams/get-team)會示範如何藉由提供特定屬性或資訊，在Microsoft Teams中尋找小組。
> - OneDrive網站和 URL：[關於OneDrive URL](/onedrive/list-onedrive-urls#about-onedrive-urls)提供使用者OneDrive URL 之適當格式和屬性的相關資訊。 使用此選項可協助您識別搜尋中OneDrive網站。

5. 在 [ **定義搜尋設定** ] 頁面上，您可以選擇選擇各種進階搜尋選項來變更預設搜尋。 在這裡，您也可以選擇先取得預估，再自動傳回資料。 如果您選擇其中任何一個選項，當您選取 [ **下一步** ] 時，將會繼續進行其他畫面。 請參閱下 [方的定義搜尋設定](#defining-search-settings) 取得詳細資料。 如果您不想變更搜尋，請將所有選項保留空白，然後選取 **[下一步** ] 以前進到下一個階段。

6. 在 [ **選取要求類型** ] 頁面上，選擇要求類型：[ **存取]**、[ **導** 出] 或 [ **已標記的清單] 以進行後續** 追蹤 ([查看上述) 的描述](#request-types) 。 指出要求是否與資料隱私權規定有關。 檢閱或變更完成期限，預設為要求建立日期後的兩周。 然後選取 **[下一步]**。

7. 在 [ **確認或編輯此要求的名稱** ] 頁面上，您可以保留或編輯為要求提供的易記名稱，然後輸入選擇性的描述。 然後選取 **[下一步]**。

8. 在 [ **檢閱並完成]** 頁面上，檢閱您在先前步驟中輸入的內容摘要。 選取每個區段中的 **[編輯] 連結** ，即可編輯任何欄位。 當您完成時，請選取 [ **建立要求]**。

建立要求之後，您會看到確認畫面。 選 **取 [完成** ] 以返回主要的 [主體權利要求] 畫面。 新的要求會列在您的要求清單頂端。 從清單中選取它，以開始檢閱並完成進度階段。

#### <a name="what-happens-after-your-request-is-created"></a>建立要求之後會發生什麼事

我們會根據您的搜尋參數，立即開始計算預期要尋找的資料量估計。 請在幾分鐘到一小時內流覽要求的詳細資料頁面，以查看估計結果，以及要求是否已移至下一個階段。 您的要求會自動從 **資料估計** 進展至 **資料擷取** ，但下列情況除外：

 - 如果您選擇在步驟 5 **定義搜尋設定** 中先接收資料 esimtae，您將能夠檢視搜尋的詳細資料，並在擷取所有資料之前變更搜尋。 如果您已在 **[資料****估計**] 中暫停，則必須手動前進到下一個資料擷取階段。
 
 - 如果您未選擇在資料估計時停止，但我們預測您的搜尋將會產生大量資料，您會在要求頂端看到一個訊息列，其中包含編輯搜尋的連結，然後再繼續進行 **資料擷取**。

若要深入瞭解這些階段，請造訪 [資料估計和擷取](subject-rights-requests-data-retrieval.md)。

## <a name="defining-search-settings"></a>定義搜尋設定

當您建立主體許可權要求時，預設搜尋會根據您在建立精靈 [ **位置** ] 頁面上的選取專案執行。 如果您想要進行更具目標的搜尋，或如果您想要在擷取內容專案之前選擇取得資料的估計值，您可以在要求建立精靈的 [ **搜尋設定** ] 頁面上進行這些選擇。

### <a name="advanced-search-options"></a>進階搜尋選項

- **精簡搜尋**：此選項可讓您指定額外的屬性，以協助識別組織資料中的資料主體。 選擇此選項之後，系統會提示您新增更多搜尋參數，如下列 [精簡搜尋](#refining-your-search)中所述。
- **包含資料主體所撰寫的內容**：此選項會尋找資料主體所撰寫的內容。 範例包括由資料主體建立或上傳至SharePoint者所建立的檔案。 選取此選項可能會大幅增加傳回的資料量。
- **包含所有版本的專案**：如果您選取SharePoint作為搜尋位置，預設搜尋查詢只會傳回目前版本的SharePoint專案。 核取此方塊會傳回目前和所有舊版的SharePoint專案，產生大量資料供您檢閱。

> [!TIP]
> 您可以選擇上傳其他資料，讓Priva根據確切的資料值來識別資料主體。 若要深入瞭解，請 [參閱主體許可權要求的資料比對](subject-rights-requests-data-match.md)。

### <a name="data-estimate"></a>資料估計

[ **取得預估第一個** ] 選項會顯示在自動擷取資料之前，預期會找到多少資料的估計值。 當您的估計值顯示在要求的詳細資料頁面上時，您可以選擇檢視搜尋結果，並預覽已探索到專案的取樣。 如果專案代表預期的結果，您必須選取 [ **擷取資料** ]，才能繼續實際擷取內容專案。 取得 [有關資料估計階段](subject-rights-requests-data-retrieval.md)的詳細資料。

## <a name="refining-your-search"></a>精簡搜尋

如果您在 [**搜尋設定**] 頁面上選擇 **[精簡搜尋**]，或是在 **[資料估計**] 階段暫停並選擇編輯搜尋查詢，系統會提示您提供個人屬性和條件的詳細資料，以進一步以搜尋結果為目標。 您可以在其中一個或兩個類別中進行新增。 當您在本節中完成選取時，要求的資料擷取將會以您的搜尋設定為基礎。

如果您已在 **[資料估計** ] 階段暫停，並選擇編輯搜尋查詢，則也會要求您提供下列詳細資料。 

#### <a name="add-personal-attributes"></a>新增個人屬性

在文字欄位中輸入資料主體名稱、昵稱、電子郵件地址和位址的值。 您可以新增其他識別碼，例如出生日期或電話號碼，而文字欄位則支援以分號分隔的多個專案。 當您完成時，請選取 [ **下一步** ] 繼續前往 [ **條件** ] 頁面。

#### <a name="conditions"></a>條件

選取 [ **新增條件]** 按鈕，在一系列條件中選擇，以進一步以搜尋為目標，包括專案名稱、寄件者和收件者名稱、個人資料類型，以及專案是否在組織外部共用。 文字欄位支援以分號分隔的多個專案。 當您完成時，請選取 [ **下一步** ] 以儲存您的搜尋設定和進度至要求類型設定。
