# <a name="review-data-for-a-subject-rights-request"></a>檢查主體權力要求的資料

建立主體權力要求之後 ([深入瞭解](subject-rights-requests-create.md)Microsoft Priva 中的) ，「主體權力要求」會使用您的輸入，以尋找組織 Microsoft 365 環境中的資料主體的相符專案。 在編譯此資料後，您就可以查看所做的結果，並視需要選擇要包含的專案和密文資訊。 這些步驟可以透過 Priva 介面由多個使用者共同合作。

## <a name="step-1-review-request-details-and-monitor-progress"></a>步驟1：複查要求詳細資料和監視進度

若要查看搜尋的初始結果，請移至 [Microsoft 365 合規性中心](https://compliance.microsoft.com/)的 [Priva] 區域，並開啟 **主體許可權要求**。 您可以在此主要頁面上找到所有已開啟的主體權力要求清單。

在清單中選取您的要求，以查看 [要求詳細資料]。 您可以在這裡深入瞭解要求的屬性、搜尋結果，以及要求的狀態。 此頁面會變成您的 hub，可在管理找到的檔案、建立報告和匯出，以及完成要求時，進行共同作業。

[要求詳細資料] 頁面上的麻將牌包括：

- **詳細資料**：有關要求的基本詳細資料，包括其期限和要求日期、描述及相關的隱私權規定。
- **進度**：時程表，指出完成的步驟和尚未完成的任何工作。
- 目前進度階段的統計資料。 此麻將牌可能會顯示資料預估摘要、搜尋中找到的專案數目，以及這些專案在 Microsoft 365 中的位置，或匯出的狀態等資訊。
- **要檢查的優先順序專案**：此麻將牌會顯示 Priva 已偵測到的任何重要專案的相關資訊。 例如，這可能包括已貼上 Microsoft 敏感度標籤的機密資訊，或具有有關可能需要密文之多個個人之資料的專案。 這會協助系統管理員知道從何處開始開始進行考核。 您可以在「優先順序類型」欄篩選收集的資料下找到優先順序專案。

### <a name="understand-progress-stages"></a>瞭解進度階段

主體權力要求經過多個階段。 當主體權力要求管理員和投稿時，會自動進行某些狀態的進度和其他階段。

因為要求可能需要一段時間或多個參與者運作，所以 Priva 會提供持續的狀態更新，以及接下來要採取的步驟。 您可以在 [主體權利要求的詳細資料] 頁面的 [ **一覽** ] 索引標籤上查看這些更新。

#### <a name="data-estimate"></a>資料預估
當您建立要求之後，Priva 會立即開始尋找對 Microsoft 365 環境中的資料主體的潛在相符專案。 一旦我們識別出我們認為符合您準則的所有專案，您會在要求的 **[一覽**] 頁面上，看到 [**資料預估摘要** 卡] 中的估計值。 搜尋範圍內的資料量會影響完成估計所需的時間長度。

您的要求會自動移至下一步的資料檢索，其中所有的內容專案都會一起收集，以供專案關係人在資料評審上進行共同作業。 在某些情況下，我們會在移至 [檢索] 之前暫停資料預估，並在繼續之前通知您後續的步驟進行。

您也可以選擇在您第一次建立主體權力要求時，自動暫停資料預估階段。 在建立過程中，選取 [**搜尋設定**] 步驟中的 [**取得預估優先**] 選項。 複查 [搜尋設定步驟](subject-rights-requests-create.md#define-search-settings)的詳細資料。

#### <a name="pause-in-data-estimate-for-large-search-results"></a>在大型搜尋結果的資料預估中暫停

Priva 會注意到，您的資料預估是否預計會傳回大量專案，以查看 () 超過10K 的專案。 預估將會暫停，這樣您就可以預覽結果，並決定是否要 [編輯搜尋查詢](subject-rights-requests-create.md#refine-your-search) 以針對更特定的位置或條件，或是繼續取得所識別的專案。  我們會在螢幕上顯示符合搜尋的專案數和資料量。 您可以在螢幕頂端的訊息列中使用下列其中一個或兩個選項：

- [ **編輯搜尋查詢** ] 按鈕會直接將您帶入要求的搜尋設定，以設定較嚴格的參數並產生新的估計值。
- 只要您的搜尋查詢不會超過300K 專案，您也會看到可供您 **檢索資料** 的選項。 這可讓您選擇不要編輯您的搜尋並繼續收集資料。

#### <a name="retrieve-data"></a>檢索資料
資料檢索階段是指所有的檔案、電子郵件、聊天、影像，以及包含資料主體個人資料的其他內容專案，並放在 Azure blob 儲存體容器中以供複查。 資料取回可能需要數分鐘或很長的時間，視資料量而定。 完成此階段後，要求會自動移至下一階段的 **審閱資料**。

#### <a name="review-data"></a>檢查資料
 在此階段，您的參與者應該查看 [ **資料收集** ] 索引標籤下的結果，並執行所有適用的工作，例如密文、套用標記，以及新增附注。 當您完成審閱時，請選取 [ **完成審閱**]。

#### <a name="generate-reports"></a>產生報告
您的報表是在此階段產生。 完成後，即可在 [ **報告** ] 索引標籤底下找到。您可以匯出您的完成檔案，以進行最後的審閱，並傳遞至提出要求的資料主體。

#### <a name="close-the-request"></a>關閉要求
關閉的要求表示所有工作已完成，可滿足此主體權力要求。 所有收集到的資料會根據您的 [資料保留設定](priva-settings.md#data-retention-periods)保留。

## <a name="step-2-optional-view-and-edit-search-queries"></a>步驟 2 (選用) ：查看和編輯搜尋查詢

若要查看有關主體權利要求背後資料搜尋的詳細資訊，請選取 [ **查看搜尋查詢詳細** 資料]。 這會開啟窗格摘要查詢，並顯示找到之內容的進一步詳細資料。

您可以在這裡選擇 **預覽搜尋結果** ，以查看會傳回此查詢的內容類型。 如果您想要變更此搜尋的屬性，但尚未開始「取得資料階段」，您可以使用 [ **編輯搜尋查詢** ] 選項。

「編輯搜尋查詢導向程式」可讓您變更或新增資料主體識別、搜尋篩選和條件的屬性，以及要在其中尋找資料 (的位置，包括 Exchange、SharePoint、OneDrive 及/或 Teams) 。 使用這些選項可達成您想要的明確程度。 您可以先查看新查詢的最後一個版本，然後再進行 **儲存**。

當您完成編輯搜尋查詢時，會執行新的搜尋以取代先前的搜尋結果。 這會將您的狀態從 [ **進度** ] 區段重設為第一個步驟（ **資料預估**）。 新的搜尋可能需要長達60分鐘才能完成。 完成後，您會在要求的 [詳細資料] 頁面上看到更新的結果。

## <a name="step-3-review-data"></a>步驟3：檢查資料

在此階段，您的參與者應該查看 [**資料收集**] 索引標籤下的結果。系統會自動設定 Teams 通道，以協助所有的專案關係人進行內容審閱。 如需詳細資訊，請參閱 [協同查看資料](#collaborate-on-data-review) 。 資料審閱步驟的基本工作如下所述。

#### <a name="mark-items-as-include-or-exclude-and-add-notes"></a>將專案標記為包含或排除並新增附注

檢查您的搜尋傳回的已識別專案清單。 如果您決定要將專案納入最後報告的一部分給資料主體，請在命令列上選取 [ **包含** 在專案清單的頂端]。 您也可以在專案清單右側的 [內容審閱] 區域中，選取 [藍色 **包含** ] 按鈕。 當您選取 [ **包含**] 時，會顯示一個彈出窗格，其中包含一個新增附注的選項。 完成後，請選取 [ **提交** ]，將專案的審閱狀態儲存為 [ **包含**]。

如果專案不屬於要求的一部分，請在命令列或 [內容審閱] 區域中的 [**排除**] 按鈕上選取 [**排除**]。 排除專案表示它不會包含在 [資料主體所產生的最終報告](subject-rights-requests-reports.md)中。

> [!NOTE]
> 如果您將專案標示為 **Exclude**，您必須將附注新增為理由，以決定其與主體權利要求不符的原因。 附注是出於內部的目的，不會包含在最終報告中。

如果內容顯示為誤報，請選取 [ **不符合** ]，然後按一下彈出窗格中的 [ **確認**]。 此巨集指令會從您的最終報告中排除該檔案，並將該專案標記為未在搜尋中偵測到的專案。

#### <a name="apply-tags"></a>套用標籤

標記可用於協助您識別需要進一步注意的專案。 Priva 提供三個預設標籤-- **執行後續操作**、 **刪除** 和 **更新** --您可以設定其描述。 Priva 也提供兩個您可以命名及描述的自訂標記。

例如，如果您在資料評審期間判斷您的組織不需要保留內容專案，則可以套用 **Delete** 標記，然後匯出所有已標記檔案的清單，這樣當您完成要求時，您可以回過頭並刪除已識別的專案。

您在 **設定** 中管理的五個標記會套用至所有的主體權力要求。

**若要新增或移除標記：**

- 從要求的 [ **資料收集** ] 索引標籤上，選取清單中的專案。
- 在清單右邊的 [專案預覽] 區域中，選取底部列的 [套用 **標記** ] 按鈕。 您也可以選取專案名稱右側的三個點，然後選取 [套用 **標記** ] 選項。
- 帶有標記清單的快顯視窗隨即出現。 選取您要套用至專案之任何標記旁邊的方塊。 取消勾選核取方塊將會移除標記。
- 當您完成選取 [ **儲存**] 時，會儲存標記選擇並關閉彈出窗格。

**若要新增自訂標記或更新標記描述：**
- 在 [主體權利要求] 頁面上，選取畫面右上角的 [**設定**]，即可取得您的 Priva 設定。
- 移至 [ **資料審閱標記** ] 頁面，然後選取標記以輸入描述，以及自訂標記的名稱。 深入瞭解 [標記設定](priva-settings.md#data-review-tags)。

**若要匯出標記的項目的清單：**
- 前往主體權利要求中所 **收集的資料** 頁面。
- 在專案清單上方，選取向下箭號圖示，當您將游標移到該專案上方時，會顯示 [ **匯出** ]。
- Excel 檔案會下載，以顯示搜尋要求所收集之所有專案的屬性。 尋找 [ **標記** ] 欄，以透過標記來識別及排序專案。

#### <a name="use-the-annotate-command-to-redact-text"></a>使用 [批註] 命令來密文文字
內容檢查區域中的 **批註** 命令，可讓您在內容專案中建立內嵌標記及密文資料。 例如，如果您需要針對也包含其他資料主體的個人資訊的個人包含檔案，您可以在命令列的 [繪圖] 按鈕下使用 **區域密文** ，以向外實心不屬於要求之人員的所有資訊。 當您編輯完成時，選取 [ **包含** ]，將 redacted 檔案新增至要求。 批註會建立檔案的複本，該複本儲存在您的 Azure blob 中。 原始檔案會保持不變，並儲存在其原始位置。

#### <a name="enter-notes-about-a-file"></a>輸入檔的附注
若要新增或查看專案上的附注，請從其列選取專案，然後移至右邊的 [內容審閱] 區域中的 [檔案 **附注** ] 索引標籤。 您也可以使用 [ **新增檔附注** ] 選項來建立新的批註。 若要在整體案例層級查看或新增附注，請移至上方的 [主要 **備** 忘] 索引標籤，然後使用 [ **新增案例附注**]。 這些附注對使用要求的使用者可以看見，但不會包含在最後一個報告中，也不會與資料主體共用。

#### <a name="complete-the-review"></a>完成評審

當所有專案都已經過檢查，而且您已將其狀態設定為 [ **包括**]、[ **排除**] 或 [ **不符合] 時，請** 選取要求內右上角的 [  **完整檢查** ] 按鈕，即可關閉「審閱」步驟。 彈出窗格會顯示資料摘要，並新增任何相關的附注。 這些附注適用于內部記錄，而且不會與資料主體共用。

在彈出窗格上選取 [ **完成複查** ]，以完成 [回顧] 步驟。 您將會在稍後的 [ **報告** ] 索引標籤下提供您的決策摘要。

### <a name="collaborate-on-data-review"></a>在資料評審時進行共同作業

Priva 支援透過 Microsoft Teams 共同作業，以允許群組一起使用主題許可權要求。 當您建立新的要求時，會自動建立 Teams 通道，並依預設與要求相關聯。 您可以在這裡討論要求並安全地分享輸入和貢獻。 若要加入交談，請開啟您的要求並使用 [ **與合作者** 互動] 選項。 這會開啟 Microsoft Teams，並將您放在您的主體權力要求小組網站的一般通道內。

若要查看可在您的小組網站上流覽及參與的活動合作者清單，請在您的主體許可權要求中開啟 [ **合作者** ] 索引標籤。若要新增此要求的其他使用者共同作業，請選取 [ **新增合作者** 的選項]。

若要在建立主體權力要求時變更產生 Teams 網站的預設行為，請移至上方流覽中的 [**設定**]，然後選取 [ **Teams** 共同作業] 以修改設定。

您也可以使用主語許可權要求的右上方的 [**共用**] 選項，透過 Teams 或電子郵件來迴圈人員，或是複製 Priva 中頁面的連結。 透過「Teams 共用」可讓您選取現有的 Teams 網站和頻道，以供您的帳戶使用，以供您在您的帳戶中張貼此案例的連結，以及您提供的任何郵件。

## <a name="step-4-close-the-request"></a>步驟4：關閉要求

當您執行所有必要的動作來解決您的主體權力要求時，請選取 **[關閉要求**]。 這會建立最終報告，可在 [ **報告]** 索引標籤上找到。完成時間可能需要一段時間，視要求中的檔案數目而定。
