# <a name="review-data-for-a-subject-rights-request"></a>檢閱主體許可權要求的資料

在您建立主體許可權要求 ([深入瞭解](subject-rights-requests-create.md)Microsoft Priva 中的) 之後，主體權利要求解決方案會使用您的輸入來尋找組織Microsoft 365環境中資料主體的相符專案。 編譯此資料之後，您可以檢閱結果、選擇要包含的專案，以及視需要修訂資訊。 這些步驟可由多位使用者透過 Priva 介面共同作業。

## <a name="step-1-review-request-details-and-monitor-progress"></a>步驟 1：檢閱要求詳細資料並監視進度

若要查看搜尋的初始結果，請移至 [Microsoft Purview 合規性入口網站](https://compliance.microsoft.com/) 的 Priva 區域，然後開啟 **[主體許可權要求]**。 您可以在此主頁面上找到所有開啟的主體許可權要求清單。

在清單中選取您的要求，以查看要求詳細資料。 您可以在這裡深入瞭解要求的屬性、搜尋結果和要求的狀態。 此頁面將成為您的中樞，可在管理找到的檔案、建立報表和匯出，以及完成要求時共同作業。

要求詳細資料頁面的 [ **概觀** ] 索引標籤會提供要求的詳細資料、顯示您目前步驟的進度指示器，以及找到之資料的重要資訊。 此頁面上的磚包含下列專案：

##### <a name="details"></a>詳細資料

**[詳細資料**] 卡片會顯示基本資訊以引導您進行要求，例如要求的期限、建立日期、描述，以及與要求相關的隱私權規定。

##### <a name="progress"></a>進度

**進度** 卡片會列出程式中的每個步驟：[資料估計]、[擷取資料]、[檢閱資料]、[產生報告] 和 [關閉要求]。 步驟旁邊的填滿藍色圓圈表示您目前使用的步驟。 藍色圓圈內的核取記號表示步驟已完成，而未填滿的圓圈表示該步驟尚未開始。

##### <a name="total-number-of-items-found"></a>找到的專案總數

目前進度階段的相關統計資料。 此圖格可能會顯示資料估計摘要、搜尋中找到的專案數目及其在Microsoft 365中的位置，或匯出狀態等資訊。

##### <a name="priority-items-to-review"></a>要檢閱的優先順序專案

[ **要檢閱的優先順序專案** ] 圖格會顯示您在開始檢閱時可能想要設定優先順序的專案。 磚會顯示屬於下列類別的專案計數：
- **機密**：這些是套用 [Microsoft 敏感度標籤](/microsoft-365/compliance/sensitivity-labels) 的專案。 例如，具有「高度機密」標籤的 Word 檔。 
- **多人資料**：這些專案包含一個以上人員的個人資料。 如果您想要將這些專案包含為最終資料封裝的一部分，則必須修訂檔案中不相關的資料。 如需詳細資訊，請參閱 [下方的步驟 3：檢閱資料](#step-3-review-data) 。 請注意，為了讓 Priva 識別具有多人資料的專案，您的組織必須設定 [主體許可權要求的資料比對](subject-rights-requests-data-match.md)。

**如何找出您的優先順序專案：**

首先，請遵循下列步驟，確定您已在資料 **收集** 的專案資料表中啟用它們的檢視：

- 在 [ **收集的資料] 索引** 標籤上，選取專案清單頂端的 [ **自訂** 資料行]。
- 在 [ **編輯資料行]** 飛出視窗窗格中，于 [ **優先順序類型**] 旁放置檢查。
- 選取 **套用**。 您的專案清單現在會有 **[優先順序類型]** 資料行。

現在您可以識別優先順序專案，並藉由排序 **[優先順序類型** ] 資料行來將類似的類型分組來尋找它們。

### <a name="understand-progress-stages"></a>瞭解進度階段

主體許可權要求會經歷多個階段。 當主體許可權要求系統管理員和參與者完成如檢閱檔案等基本步驟時，某些狀態會自動進行，而其他階段則會繼續進行。

由於要求可能需要隨著時間或由多個參與者處理，因此 Priva 會提供狀態的持續更新，以及後續要採取之步驟的指引。 您可以在主體許可權要求詳細資料頁面的 [ **概觀** ] 索引標籤上檢視這些更新。

#### <a name="data-estimate"></a>資料估計
建立要求之後，Priva 會立即開始尋找Microsoft 365環境中資料主體的潛在相符專案。 一旦我們識別出所有我們認為符合您準則的專案，您就會在要求的 [**概觀**] 頁面上，于 **[資料估計摘要**] 卡片中看到估計值。 搜尋範圍內的資料量會影響完成估計所需的時間長度。

您的要求會自動移至資料擷取的下一個階段，其中所有內容專案會一起收集，讓專案關係人可以共同進行資料檢閱。 在某些情況下，我們會先暫停資料估計，再繼續進行擷取，並通知您後續要採取的步驟，再繼續進行。

當您第一次建立主體許可權要求時，也可以選擇在資料預估階段自動暫停。 在建立程式期間，選取 [**搜尋設定**] 步驟期間的 [**取得估計第一個**] 選項。 檢閱 [搜尋設定步驟的詳細資料](subject-rights-requests-create.md#define-search-settings)。

#### <a name="pause-in-data-estimate-for-large-search-results"></a>大型搜尋結果的資料估計暫停

Priva 會注意到，如果您的資料預估預計會傳回大量的專案，以檢閱 (超過 100000 個專案) 。 預估將會暫停，讓您可以預覽結果，並決定是否 [要編輯搜尋查詢](subject-rights-requests-create.md#refine-your-search) 以以更特定的位置或條件為目標，或繼續擷取已識別的專案。  我們會在螢幕上顯示符合您搜尋的專案數目和資料量。 您在畫面頂端的訊息列中會有下列其中一個或兩個選項：

- [ **編輯搜尋查詢** ] 按鈕會直接帶您進入要求的搜尋設定，以設定更嚴格的參數並產生新的估計值。
- 只要您的搜尋查詢不是超過 30 萬個專案，您也會看到 [ **擷取資料**] 的選項。 這可讓您選擇不要編輯搜尋，並繼續收集資料。

#### <a name="retrieve-data"></a>擷取
資料擷取階段是擷取包含資料主體個人資料的所有檔案、電子郵件、聊天、影像和其他內容專案，並放在 Azure Blob 儲存體容器中以供檢閱。 視資料量而定，資料擷取可能需要幾分鐘或更長的時間。 當此階段完成時，要求會自動移至 [ **檢閱資料**] 的下一個階段。

#### <a name="review-data"></a>檢閱資料
 在這個階段，您的參與者應該檢閱 [ **收集的資料** ] 索引標籤底下的結果，並執行所有適用的工作，例如修訂、套用標籤和新增附注。 當您完成檢閱時，請選取 [ **完成檢閱]**。

#### <a name="generate-reports"></a>產生報告
您的報表會在此階段產生。 完成時，可以在 [ **報** 表] 索引標籤下找到這些專案。您已完成的檔案可以匯出以進行最終檢閱，並傳遞給提出要求的資料主體。

#### <a name="close-the-request"></a>關閉要求
已關閉的要求表示已完成所有工作，以完成此主體許可權要求。 所有收集的資料和報表都會根據您的 [資料保留設定保留](priva-settings.md#data-retention-periods)。

## <a name="step-2-optional-view-and-edit-search-queries"></a>步驟 2 (選用) ：檢視和編輯搜尋查詢

若要查看主體許可權要求背後的資料搜尋詳細資訊，請選 **取 [檢視搜尋查詢詳細資料]**。 這會開啟摘要查詢的窗格，並顯示找到之內容的進一步詳細資料。

您可以在這裡選擇 **[預覽搜尋結果** ]，以查看此查詢將傳回何種類型的內容。 如果您想要變更此搜尋的屬性，但尚未開始擷取資料階段，您可以使用 **[編輯搜尋查詢** ] 選項。

編輯搜尋查詢引導式程式可讓您變更或新增資料主體識別、搜尋篩選和條件的屬性，以及要在其中尋找資料 (的位置，包括Exchange、SharePoint、OneDrive和/或Teams) 。 使用這些選項來達到您想要的特定性層級。 您可以先檢閱新查詢的最終版本，再按 [ **儲存]**。

當您完成編輯搜尋查詢時，將會執行新的搜尋來取代先前的搜尋結果。 這會將您在 [ **進度** ] 區段中的狀態重設為第一個步驟 **[資料估計]**。 新的搜尋可能需要 60 分鐘才能完成。 完成後，您會在要求的詳細資料頁面上看到更新的結果。

## <a name="step-3-review-data"></a>步驟 3：檢閱資料

在這個階段，您的參與者應該在 [**收集的資料] 索** 引標籤下檢閱結果。系統會自動設定Teams通道，以協助所有專案關係人檢閱內容。 如需詳細資訊，請參閱 [在資料檢閱上共同](#collaborate-on-data-review) 作業。 資料檢閱步驟的基本工作如下所述。

#### <a name="mark-items-as-include-or-exclude-and-add-notes"></a>將專案標示為 [包含] 或 [排除] 並新增附注

檢閱搜尋所傳回的已識別專案清單。 如果您決定將專案納入做為最後一份報表的一部分，請回到資料主體，請選取專案清單頂端命令列上的 [ **包含** ]。 您也可以選取專案清單右側內容檢閱區域中的藍色 [ **包含** ] 按鈕。 當您選取 [ **包含**] 時，飛出視窗窗格隨即出現，其中包含新增附注的選項。 當您完成時，請選取 **[提交** ]，將專案的檢閱狀態儲存為 **[包含]**。

如果專案不屬於要求的一部分，請選取命令列上的 [ **排除** ] 或內容檢閱區域中的 [ **排除** ] 按鈕。 排除專案表示不會包含在 [為數據主體產生的最終報表中](subject-rights-requests-reports.md)。

> [!NOTE]
> 如果您將專案標示為 **[排除**]，則必須新增附注，作為其不與主體許可權要求有關的理由。 附注是供內部使用，不會包含在最終報告中。

如果內容顯示為誤判，請選取 **[不相符]** ，然後在飛出視窗窗格中選取 [ **確認]**。 此動作會從最終報表中排除檔案，並將專案標示為搜尋中不應該偵測到的專案。

#### <a name="apply-tags"></a>套用標籤

標籤可用來協助您識別需要進一步注意的專案。 Priva 提供三個預設標籤 ， **追蹤**、 **刪除** 和 **更新** -- 您可以為其設定描述。 Priva 也提供兩個您可以命名和描述的自訂標記。

例如，如果您在資料檢閱期間判斷您的組織不需要保留內容專案，您可以套用 **Delete** 標籤，然後匯出所有已標記檔案的清單，以便在完成要求時返回並刪除已識別的專案。

您在 設定 中管理的五個標籤 **會** 套用至所有主體許可權要求。

**若要新增或移除標記：**

- 從要求之 [ **收集** 的資料] 索引標籤上的清單中選取專案。
- 在清單右側的專案預覽區域中，選取底部資料列上的 [ **套用標籤** ] 按鈕。 您也可以選取專案名稱右邊的三個點，然後選取 [ **套用標籤]** 選項。
- 隨即會出現具有標籤清單的飛出視窗窗格。 核取您要套用至專案的任何標籤旁的方塊。 取消核取核取方塊將會移除標記。
- 當您完成時，請選取 [ **儲存**]，這會儲存您的標籤選取專案，並關閉飛出視窗窗格。

**若要新增自訂標籤或更新標籤描述：**
- 從 [主體許可權要求] 頁面，選取畫面右上角的 **[設定**]，以取得您的 Priva 設定。
- 移至 **[資料檢閱標籤]** 頁面，然後選取要輸入描述的標籤，並針對自訂標籤選取名稱。 深入瞭解 [標籤設定](priva-settings.md#data-review-tags)。

**若要匯出標記的項目的清單：**
- 移至主體許可權要求中的 **[收集** 的資料] 頁面。
- 在專案清單上方，選取向下箭號圖示，當您將滑鼠停留在專案上時，會顯示 [ **導** 出]。
- Excel檔案將會下載，其中會顯示搜尋要求所收集之所有專案的屬性。 尋找 [ **標記]** 資料行，以依標籤識別和排序專案。

#### <a name="use-the-annotate-command-to-redact-text"></a>使用 Annotate 命令來修訂文字
內容檢閱區域中的 **Annotate** 命令可讓您在內容專案內建立內嵌標記和修訂資料。 例如，如果您需要包含也包含不同資料主體個人資訊的個人檔案，您可以在命令列的 [繪圖] 按鈕下使用 **[區域修訂** ]，以清除與提出要求之人員無關的所有資訊。 當您的編輯完成時，請選取 [ **包含** ] 將已修訂的檔案新增至要求。 批註會建立檔案的複本，該檔案會儲存在您的 Azure Blob 中。 原始檔案會保持不變，並儲存在其原始位置。

#### <a name="enter-notes-about-a-file"></a>輸入檔案的相關附注
若要新增或檢閱專案上的附注，請從專案列中選取專案，然後移至右側內容檢閱區域中的 [ **檔案筆記** ] 索引標籤。 您也可以使用 **[新增檔案注意事項** ] 選項來建立新的批註。 若要檢閱或新增整體案例層級的附注，請移至上方的主要 **[附注]** 索引標籤，然後使用 **[新增案例附注]**。 處理要求的使用者會看到這些附注，但不會包含在最終報表中，或是與資料主體共用。

#### <a name="complete-the-review"></a>完成檢閱

當所有專案都經過檢閱，且您將其狀態設定為 [ **包含**]、[ **排除**] 或 [ **不相符** ] 時，就可以選取要求右上角的 [  **完成檢閱** ] 按鈕來關閉檢閱步驟。 飛出視窗窗格會顯示資料摘要，並新增任何相關的附注。 這些附注適用于內部記錄保留，且不會與資料主體共用。

在飛出視窗窗格上選取 [ **完成檢閱** ]，以完成檢閱步驟。 稍後會在 [ **報** 表] 索引標籤下提供您決策的摘要。

### <a name="collaborate-on-data-review"></a>在資料檢閱上共同作業

Priva 透過Microsoft Teams支援共同作業，讓您的群組能夠共同處理主體許可權要求。 當您建立新的要求時，預設會自動建立Teams通道並與您的要求相關聯。 您可以在這裡討論要求，並安全地共用輸入和參與。 若要加入對話，請開啟您的要求，並使用 **[與共同作業者聊天** ] 選項。 這會開啟Microsoft Teams，並將您放在主體許可權要求小組網站的 [一般] 頻道內。

若要檢閱可檢視並參與小組網站的作用中共同作業者清單，請在您的主體許可權要求中開啟 [ **共同作業者] 索引卷** 標。若要新增其他使用者以在此要求上共同作業，請選取 [ **新增共同作業者]** 選項。

若要在建立主體許可權要求時變更產生Teams網站的預設行為，請移至頂端導覽中的 **設定**，然後選 **取Teams共同作業** 來修改設定。

您也可以在主體許可權要求內使用右上方的 **[共用**] 選項，透過Teams或電子郵件將人員迴圈至 ，或是將連結複製到 Priva 中的頁面。 透過Teams共用可讓您選取您帳戶可用的現有Teams網站和通道，其中會張貼此案例的連結以及您提供的任何訊息。

## <a name="step-4-close-the-request"></a>步驟 4：關閉要求

當您已執行所有必要的動作來解決您的主體許可權要求時，請選取 **[關閉要求]**。 這會建立最終報表，可在 [報表] 索引卷 **標** 上找到。視要求中的檔案數目而定，完成可能需要一些時間。
